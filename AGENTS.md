# AGENTS.md

> Файл для AI-агентов (Cursor, Claude Code и других). Прочитай **до** внесения любых изменений.

## 1. Жёсткие правила

1. **Spec-Driven Development.** Код без спецификации не пишется. Для фичи должны быть `specs/NNN-slug/spec.md`, `plan.md`, `tasks.md`. Если их нет — вызови `/specify`.
2. **Конституция.** Соблюдай [`.specify/memory/constitution.md`](.specify/memory/constitution.md). Все 12 принципов обязательны.
3. **pnpm only.** `npm`/`yarn` запрещены.
4. **Контент — JSON.** Защитники, враги, уровни, товары, достижения — только в `content/**/*.json`, валидируются Zod-схемами из `packages/shared`. Никакого хардкода статов в коде.
5. **Shared types.** Всё, что пересекает границу клиент↔сервер или код↔контент, живёт в `packages/shared`.
6. **Pointer Events для ввода.** Никаких раздельных веток mouse/touch.

## 2. Рабочий поток

Используй slash-команды из `.cursor/commands/`:

| Команда | Что делает |
|---------|------------|
| `/constitution` | Показывает принципы проекта. |
| `/specify` | Создаёт `specs/NNN-slug/spec.md` из идеи. |
| `/plan` | Создаёт `plan.md` + вспомогательные файлы. |
| `/tasks` | Разбивает план на чек-лист задач. |
| `/implement` | Выполняет задачи (agent mode). |

## 3. Структура монорепо

- `apps/web/` — Phaser 3 + TS клиент.
- `apps/api/` — Fastify + Prisma бэкенд.
- `packages/shared/` — Zod-схемы и DTO.
- `content/` — JSON контент игры.
- `docs/plans/` — мастер-планы.
- `.specify/` — конституция и шаблоны.
- `specs/` — спецификации фич.
- `.cursor/commands/` — slash-команды.

## 4. Чего НЕ делать

- Не обходить принципы конституции без обсуждения.
- Не добавлять функционал, которого нет в активной `spec.md`.
- Не создавать файлы вне структуры (например, `src/` в корне).
- Не пропускать Constitution Check в `plan.md`.
- Не менять баланс (HP, урон) в коде — только в JSON-контенте.
- Не использовать `npm install` или `yarn add`.
