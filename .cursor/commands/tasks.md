# /tasks

Разбей план на пошаговый чек-лист задач.

## Шаг 1. Найди активную фичу

1. Если пользователь указал `NNN-slug` — используй её.
2. Иначе — последняя, у которой есть `plan.md`, но нет `tasks.md` или он пустой.
3. Убедись, что `specs/NNN-slug/plan.md` существует и помечен `Approved`. Если `Draft` — предупреди пользователя.

## Шаг 2. Загрузи контекст

1. `.specify/memory/constitution.md`
2. `specs/NNN-slug/spec.md`
3. `specs/NNN-slug/plan.md`
4. `specs/NNN-slug/data-model.md` (если есть)
5. `specs/NNN-slug/quickstart.md`
6. [`.specify/templates/tasks-template.md`](../../.specify/templates/tasks-template.md)

## Шаг 3. Сгенерируй `tasks.md`

В `specs/NNN-slug/tasks.md`:

Порядок задач строго такой:
1. **Setup** — типы, Zod-схемы в `packages/shared`.
2. **Contract** — API-роуты, e2e-тесты для них.
3. **Content** — JSON-файлы, генерация/оптимизация спрайтов.
4. **Core** — сущности и системы (`entities/*`, `systems/*`).
5. **UI** — сцены и компоненты (`scenes/*`, `ui/*`).
6. **Integration** — проверка AC вручную (десктоп + мобилка).
7. **Deploy** — сборка, PR, CI, staging.

Правила:
- Каждая задача — атомарная (≤ 1–2 часа), с чёткой командой/действием.
- Указывай в скобках к какому AC относится: `(AC-1, AC-2)`.
- Нумерация сквозная: `T-001`, `T-002`...
- В конце — блок «Прогресс» с подсчётом.

## Шаг 4. Отчитайся

Перечисли созданные задачи по группам. Предложи:
1. Проверить список.
2. Вызвать `/implement` для начала выполнения.

## Что НЕ делать

- Не пиши код.
- Не выполняй задачи сейчас.
