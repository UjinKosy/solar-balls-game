# /specify

Создай новую спецификацию фичи по запросу пользователя. Следуй строгой процедуре:

## Шаг 1. Загрузи контекст

1. Прочитай [`.specify/memory/constitution.md`](../../.specify/memory/constitution.md) — принципы проекта.
2. Прочитай [`docs/plans/solar-balls-plan.md`](../../docs/plans/solar-balls-plan.md) — мастер-план с этапами.
3. Прочитай [`.specify/templates/spec-template.md`](../../.specify/templates/spec-template.md) — скелет.

## Шаг 2. Определи номер и slug

1. Посмотри в [`specs/`](../../specs/) существующие папки вида `NNN-slug`.
2. Выбери следующий номер (например `001`, `002` — три цифры с ведущими нулями).
3. Slug — короткий kebab-case из 2–4 слов, отражающий суть фичи (`main-menu`, `grid-and-drag-drop`, `guest-auth-backend`).
4. Создай папку `specs/NNN-slug/`.

## Шаг 3. Выясни намерение

Задай пользователю через AskQuestion:

1. **К какому этапу плана** относится фича (0..13)?
2. **Одна или несколько** спек будет на этот этап?
3. **Что не входит** в скоуп — чтобы не расползалось.

Если пользователь уже сказал достаточно — не спрашивай лишнего.

## Шаг 4. Сгенерируй spec.md

1. Скопируй шаблон `spec-template.md` в `specs/NNN-slug/spec.md`.
2. Заполни все разделы на основе запроса пользователя:
   - Контекст и мотивация (зачем сейчас).
   - Пользовательские сценарии (As-a / I-want / So-that).
   - Функциональные требования (FR-N).
   - Нефункциональные требования (NFR-N).
   - Критерии приёмки (AC-N, проверяемые).
   - Ключевые сущности (без полей БД).
   - Demo (как будет выглядеть).
   - Out of scope.
   - Открытые вопросы.
3. **Не пиши технологии** (Phaser, Fastify, TS, файлы). Всё это — в `plan.md`.
4. Пройди по чек-листу принципов в разделе «Проверка соответствия Конституции».

## Шаг 5. Отчитайся

Выведи пользователю:
- Путь к созданному `spec.md`.
- Список открытых вопросов, если есть.
- Что делать дальше: «Отредактируй по вкусу, затем вызови `/plan`».

## Что НЕ делать

- Не создавай сразу `plan.md` или `tasks.md` — только спека.
- Не пиши код.
- Не упоминай имена файлов исходников и фреймворков в `spec.md`.
