# /implement

Начни выполнение задач из `tasks.md` текущей фичи. **Требуется agent mode.** Если сейчас plan mode — попроси пользователя переключить.

## Шаг 1. Найди активную фичу

1. Если пользователь указал `NNN-slug` — используй её.
2. Иначе — последняя с `tasks.md`, где есть незакрытые задачи.

## Шаг 2. Загрузи контекст

1. `.specify/memory/constitution.md`
2. `specs/NNN-slug/spec.md`
3. `specs/NNN-slug/plan.md`
4. `specs/NNN-slug/tasks.md`
5. `specs/NNN-slug/quickstart.md`
6. `specs/NNN-slug/data-model.md` (если есть)

## Шаг 3. Подготовка ветки

1. Проверь `git status`. Если есть несохранённые изменения — предложи закоммитить.
2. Проверь текущую ветку. Если не `feature/NNN-slug` — создай её: `git checkout -b feature/NNN-slug`.

## Шаг 4. Выполнение задач

Выполняй задачи **строго по порядку** (Setup → Contract → Content → Core → UI → Integration → Deploy).

Для каждой задачи:
1. Прочитай, что она требует.
2. Выполни действие (код, файл, команда).
3. Запусти проверку (тесты/линт/билд там, где применимо).
4. Обнови `tasks.md`: поставь галочку `[x]`.
5. Закоммить: `git commit -m "T-NNN: краткое описание"`.

Между группами задач (после Setup, Contract, Content, Core, UI, Integration) сделай **паузу** и покажи пользователю прогресс.

## Шаг 5. Проверка AC

Когда все задачи из Integration выполнены — пройди по `spec.md § Acceptance Criteria` и проверь каждый пункт. Отметь прохождение в самом файле или в `tasks.md`.

## Шаг 6. Deploy

1. `pnpm build` должен проходить.
2. Создай PR: `gh pr create` (если настроен) — в теле PR укажи ссылки на `spec.md`, `plan.md`, `tasks.md`.
3. Дождись прохождения CI.
4. Проверь фичу на staging URL.

## Шаг 7. Закрытие

1. Обнови статусы файлов `spec.md`, `plan.md` → `Implemented`.
2. Обнови todo-прогресс.
3. Отчитайся пользователю.

## Правила

- Не пропускай Constitution Check. Если задача противоречит принципу — стоп, обсуди.
- Не добавляй функционал, которого нет в `spec.md` и `tasks.md`. Расползание скоупа — в открытые вопросы.
- Если задача провалилась — пометь блокер в `tasks.md` и сообщи пользователю, не продолжай слепо.
