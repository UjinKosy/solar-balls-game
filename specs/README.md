# `specs/` — спецификации фич

Здесь хранятся **все спецификации** проекта, по одной папке на фичу.

## Структура

```
specs/
├── 001-monorepo-and-deploy/
│   ├── spec.md
│   ├── plan.md
│   ├── tasks.md
│   ├── quickstart.md
│   ├── data-model.md        # опционально
│   ├── research.md          # опционально
│   └── contracts/           # опционально, Zod-схемы API
├── 002-main-menu/
│   └── ...
└── ...
```

## Правила нейминга

- `NNN-slug`, где `NNN` — трёхзначный номер по порядку, `slug` — kebab-case из 2–4 слов.
- Номер никогда не переиспользуется; удалённая фича оставляет «дыру».

## Жизненный цикл спеки

1. **Draft** — создаётся через `/specify`, заполняется.
2. **In Review** — готова к обсуждению; добавляется в PR.
3. **Approved** — принята, можно делать `/plan` → `/tasks` → `/implement`.
4. **Implemented** — реализована, смерджена в `main`.

## Связь с мастер-планом

Каждая спека ссылается на этап из [`../docs/plans/solar-balls-plan.md`](../docs/plans/solar-balls-plan.md) в шапке. Один этап плана обычно = одна или две спеки.
