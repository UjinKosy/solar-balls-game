# Quickstart: Монорепо и деплой-каркас

**ID фичи:** 001-monorepo-and-deploy

> Минимальная инструкция, как локально увидеть фичу в работе.

---

## Предусловия

- Репозиторий склонирован локально.
- Зависимости установлены через `pnpm install`.
- Docker Engine запущен.
- Для этапа 0 используется полный набор ассетов из `assets/*`, дополнительных изображений не требуется.

## Запуск

```bash
docker compose up -d postgres
pnpm dev
```

Открыть клиент по локальному URL разработки (из вывода `pnpm dev`).

## Сценарий проверки

1. Открыть локальный URL клиента.
2. Убедиться, что отображается пустой экран с текстом «Шаранутые игры».
3. Проверить серверный endpoint состояния:

```bash
curl -i http://localhost:3000/health
```

4. Убедиться, что сервер возвращает успешный HTTP-статус и JSON со статусом `ok`.
5. Открыть публичный URL демо и повторно убедиться, что стартовый экран отображается.

## Ожидаемый результат

Все пункты из раздела «Критерии приёмки» в `spec.md` выполняются: демо доступно локально и публично, сервер диагностируется через `GET /health`, инструкция воспроизводима для нового участника.
